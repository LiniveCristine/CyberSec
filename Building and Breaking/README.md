# 🛡️ Web Development for Cybersecurity

Este repositório é destinado ao **estudo de desenvolvimento web com foco em segurança ofensiva**.

A proposta é entender **como aplicações web são construídas** para depois compreender **como elas podem ser exploradas**.

O objetivo é desenvolver uma mentalidade de **análise profunda de sistemas**, essencial para áreas como:

- Pentest
- Bug Bounty
- Application Security (AppSec)

---

# 🧠 Metodologia: Building & Breaking

> **"You can’t hack what you don’t understand."**

A metodologia **Build & Break** consiste em duas etapas principais:

### 🏗️ Build (Construir)

Primeiro você aprende **como o sistema funciona internamente**.

Isso inclui entender:

- Arquitetura da aplicação
- Fluxo de dados
- Lógica de negócio
- Pontos de confiança
- Validação de dados
- Comunicação entre componentes

---

### 💥 Break (Quebrar)

Depois de entender como o sistema funciona, o próximo passo é **identificar e explorar falhas**.

Ou seja:

```

Entender como funciona
↓
Descobrir como pode falhar

```

Essa abordagem desenvolve uma **mentalidade ofensiva real**, utilizada por **pentesters e bug hunters**.

---

# 🚀 Vantagens da Metodologia Build & Break

## 🔍 Entendimento Profundo de Sistemas

Quando você aprende **como algo foi construído**, passa a compreender melhor:

- Arquitetura da aplicação
- Fluxo de informações
- Relações de confiança
- Regras de negócio
- Pontos críticos de validação

Isso permite identificar **vulnerabilidades complexas** que scanners automáticos não conseguem detectar.

---

## 🎯 Muito Mais Eficiente para Bug Bounty

Em **programas de Bug Bounty**, as vulnerabilidades raramente são simples.

Geralmente envolvem problemas como:

- Falhas de lógica de negócio
- Bypass de autenticação
- Escalonamento de privilégios
- Falhas em APIs
- Problemas de autorização

Esses tipos de vulnerabilidade **não aparecem em scanners automáticos**.

Elas são encontradas apenas quando você **entende profundamente como o sistema foi construído**.

# 🧪 Vulnerabilidades que Serão Exploradas

Após estudar as bases de desenvolvimento, o próximo passo será **explorar vulnerabilidades em aplicações criadas por mim mesmo**.

Entre elas:

- **SQL Injection**
- **Cross-Site Scripting (XSS)**
- **Broken Access Control**
- **CSRF (Cross-Site Request Forgery)**
- **Shell Upload**
- **JWT Improper Validation**
- **BOLA (Broken Object Level Authorization)**
- **NoSQL Injection**
- **SSTI (Server-Side Template Injection)**
